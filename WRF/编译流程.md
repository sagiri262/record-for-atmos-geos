# 编译WRF

注：本文只关注于怎么编译，不关注怎么跑通程序

还有可以参考一下立正大学(RISSHO UNIVERSITY)增田耕一的文章：https://macroscope.world.coocan.jp/ja/edu/ris/seminar/wrf/wrf.html

## 系统依赖

`sudo apt update

sudo apt install -y \
    build-essential gfortran gcc g++ make m4 csh perl tcsh \
    curl wget git vim \
    openmpi-bin libopenmpi-dev \
    libnetcdf-dev libnetcdff-dev \
    libhdf5-dev zlib1g-dev libpng-dev libjasper-dev`

1、必要的组件 build-essential
2、需要 C 和 fortran 的编译器 gfortran gcc g++ make m4 csh perl tcsh
3、下载数据需要我们使用 wget、curl 等依赖
4、并行计算我们需要 openMPI
5、WRF 需要 netCDF 格式的数据，程序完成后 WRF 会输出图片以及其他格式的数据，我们也需要使用 NCL 或 Python 把结果可视化出来，所以我们就下载 libhdf5-dev zlib1g-dev libpng-dev libjasper-dev 等依赖

**对 WPS + GRIB2，需要 zlib、libpng、jasper；并且 WPS v4.4+ 可以在 configure 时直接内部构建这些库，所以即使系统库不全，也可以走内部构建。**

现在对如果安装 zlib 、 jasper 等依赖下载出现 Error，我们有解决办法

## 编译 WRF

进入 WRF：
`cd ~/WRF/WRF
./clean -a
./configure`
然后你能看到密密麻麻的选项。所以该怎么选呢？

### 使用个人主机PC

个人主机我的建议是：要不选择新款的AMD CPU，因为在超线程上有优势，再一个在同样核数上AMD也更占优势；要不选择E5核数多的CPU，intel在科学计算领域深耕时间远超AMD这些年的努力。如果有条件，我非常建议搭建一个属于自己的洋垃圾工作站。X99主办搭配32G内存和2T左右的硬盘，加上一颗200元上下的E5 26**系列，核数大约在16～20个上下

### 使用HPC节点计算

这基本取决于你的 HPC 所分配到的节点。



编译时用：
`./compile em_real >& log.compile`
编译完成后检查：

`ls -lh main/*.exe`

官方说明编译成功后要检查 main/ 下的可执行文件。你最终至少要看到类似：

## 编译 WPS

编译 WPS 就更简单了
`cd ~/dir/WPS
./clean -a
./configure`

选择与 WRF 相同的编译器即可。

然后编译：
`./compile >& log.compile`

成功后检查：
`ls -lh geogrid.exe ungrib.exe metgrid.exe
ls -lh geogrid/src/geogrid.exe ungrib/src/ungrib.exe metgrid/src/metgrid.exe`
官方要求这 3 个程序存在，且不是 0 字节。

## 下载静态地理数据

WRF 官方说明：真实个例必须准备静态地理数据，解压后重命名为 WPS_GEOG，并在 namelist.wps 的 geog_data_path 指向该目录；完整数据展开后大约 10 GB。

`mkdir -p ~/WRF/geog`

## 准备 GFS 数据

准备下面的脚本（尽量不要放在 WPS 或者 GFS 的目录下，这会出现连链接错误，很麻烦）

`#!/usr/bin/env bash
set -euo pipefail

# ===== 用户参数 =====
DATE="20260311"          # YYYYMMDD
CYCLE="00"               # 00 / 06 / 12 / 18
FSTART=0                 # 起始预报时效
FEND=72                  # 结束预报时效
FSTEP=3                  # 间隔，WRF常用3h
OUTDIR="$HOME/WRF/data/GFS/${DATE}_${CYCLE}"

# 两个常用入口，主站失败可切换
BASE1="https://nomads.ncep.noaa.gov/pub/data/nccf/com/gfs/prod"
BASE2="https://ftp.ncep.noaa.gov/data/nccf/com/gfs/prod"

mkdir -p "$OUTDIR"
cd "$OUTDIR"

download_one() {
    local fhour="$1"
    local fff
    printf -v fff "%03d" "$fhour"

    local file="gfs.t${CYCLE}z.pgrb2.0p25.f${fff}"
    local path="gfs.${DATE}/${CYCLE}/atmos/${file}"

    echo ">>> downloading ${file}"

    # -c 断点续传
    if ! wget -c "${BASE1}/${path}"; then
        echo ">>> primary failed, try backup"
        wget -c "${BASE2}/${path}"
    fi
}

for ((fh=FSTART; fh<=FEND; fh+=FSTEP)); do
    download_one "$fh"
done

echo "Done. Files saved in: $OUTDIR"`

官方教程里用的是 GFS/FNL 类 GRIB 数据流程，WPS 的 ungrib 需要用link_grib.csh 链接 GRIB 文件，执行：
`ln -sf ungrib/Variable_Tables/Vtable.GFS Vtable`

运行 ungrib.exe，
再跑 metgrid.exe。

## 跑通 WPS

修改 namelist.wps

核心字段主要包括：
max_dom
start_date
end_date
interval_seconds
geog_data_path
prefix='FILE'
fg_name='FILE'

执行一下命令：
`cd ~/WRF/WPS

./geogrid.exe >& log.geogrid

./link_grib.csh ~/WRF/data/GFS/20260311_00
ln -sf ungrib/Variable_Tables/Vtable.GFS Vtable
./ungrib.exe >& log.ungrib

./metgrid.exe >& log.metgrid`

输出应当包括：
geo_em.d01.nc ...

FILE:YYYY-MM-DD_hh

met_em.d01.YYYY-MM-DD_hh:mm:ss.nc

## 跑 WRF

进入真实算例，执行一下命令：
`cd ~/WRF/WRF/test/em_real
ln -sf ~/WRF/WPS/met_em* .`

官方说明可以在 WRF/run 或 WRF/test/em_real 下运行；真实资料流程通常很多人直接用 test/em_real。

修改 namelist.input，重点包括：

run_days/run_hours
start_year/month/day/hour
end_year/month/day/hour
interval_seconds
max_dom
e_we, e_sn, e_vert
dx, dy
物理方案（微物理、边界层、长短波、陆面过程、积云等）
num_metgrid_levels
num_metgrid_soil_levels

紧接着直接执行

`./real.exe`

弹出结果后就可以直接执行 ./wrf.exe 了。但是这里我们是单线程运算，运算速度肯定是很慢的。我们可以执行下面的命令，就可以并行计算了，

`mpirun -np 8(核数) ./real.exe
mpirun -np 8 ./wrf.exe`

### 各目录用途
*~/WRF/WRF*
只放 WRF 源码与编译结果。
不要把不同实验的输出直接长期堆在这里。

*~/WRF/WPS*
只放 WPS 源码与编译结果。

*~/WRF/geog/WPS_GEOG*
静态地理数据目录。
namelist.wps 里 geog_data_path 指向这里。官方也明确要求这么做。

*~/WRF/data/GFS/...*
放原始 GFS GRIB2 文件。建议按日期 cycle 分目录。

*~/WRF/case/某个实验/*
每个实验独立：

wps_run/：跑 geogrid/ungrib/metgrid
real_run/：跑 real.exe/wrf.exe
output/：保存 wrfout*, wrfrst*, rsl.*
post/：后处理结果

## 关于 WRF 的 ./configur 选项

*个人机*
第一次装、第一次跑：先选 serial，确认能跑通后：再改成 dmpar，一般不建议优先选 smpar，也不建议一开始就 dm+sm，而且本个人机也不建议开 dm + sm，这样容易出问题，包括后面编译也会出问题，个人猜测可能是CPU核数不够导致的

*HPC*
绝大多数情况选 dmpar，只有在集群管理员明确建议、你也清楚节点拓扑和 OpenMP 绑定策略时，才尝试 dm+sm。smpar 在 HPC 上通常不是首选

8. link_grib.csh + Vtable.GFS + ungrib.exe
9. metgrid.exe
10. 进入 WRF/test/em_real
11. 链接 met_em*
12. 修改 namelist.input
13. real.exe
14. wrf.exe
